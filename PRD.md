## 一、产品概述

### 1\.1 产品定位

一个轻量化多平台热搜聚合网站，一站式聚合微博热搜、知乎热榜、B站热搜三大平台实时榜单。用户无需切换平台即可快速浏览全网热点，零登录、无广告、访问便捷。同时作为全栈开发学习实践项目，适用于前端、接口开发、缓存优化、异常处理技术落地练习。

### 1\.2 目标用户

- **普通用户**：上班族、学生，利用碎片化时间快速查看全网热点资讯

- **技术学习者**：前端、全栈开发人员，用于前后端分离、接口聚合、响应式布局、服务缓存等场景实战学习

### 1\.3 核心价值

整合多平台热点信息，降低用户信息获取成本；页面简洁高效、信息密度贴合主流资讯产品，同时具备完整的产品架构与迭代能力，适合项目实战落地。

---

## 二、整体架构与技术栈

### 2\.1 整体架构

前后端分离架构

- **前端**：页面渲染、响应式适配、交互展示、榜单可视化

- **后端**：数据抓取、数据格式化、接口提供、缓存管理、异常容错

### 2\.2 技术栈

- **前端**：React \+ TypeScript \+ Vite \+ 原生CSS（MVP基础版支持纯HTML/CSS/JS）

- **后端**：Node\.js \+ Express

- **部署**：前端Vercel、后端Railway，支持HTTPS公网访问

- **代码托管**：GitHub

---

## 三、核心功能（MVP 必做）

### 3\.1 首页卡片网格功能

#### 3\.1\.1 页面结构

页面由顶栏、主体卡片区、页脚三部分组成。

#### 3\.1\.2 顶部导航栏

- 展示站点名称：今日热搜

- 展示简短介绍文案：一站式查看全网各平台实时热点

- 样式：白底浅阴影通栏布局，简洁轻量化

#### 3\.1\.3 主体榜单卡片

固定展示3个平台卡片：微博热搜、知乎热榜、B站热搜

**响应式布局规则**：

- 网页端：三列卡片均等展示

- 手机端：单列卡片自适应展示

**单卡片组成结构**：

- 平台名称、榜单名称

- Top10 榜单列表

**单条榜单字段**：

- 排名序号（1\-10）

- 热点标题（可点击跳转，初始链接 \# 占位）

- 模拟热度数据

#### 3\.1\.4 页脚模块

展示项目说明、数据来源声明、非商用学习项目声明。

### 3\.2 后端接口能力

#### 3\.2\.1 通用规范

- 请求方式：GET

- 返回格式：统一JSON结构

#### 3\.2\.2 接口列表

- `GET /api/hot/:source`：获取单平台热搜榜单
source 枚举值：weibo / zhihu / bilibili

- `GET /api/hot`：一次性获取全部平台榜单数据

#### 3\.2\.3 统一返回数据格式

```json
{
  "code": 200,
  "msg": "请求成功",
  "data": [
    {
      "platform": "平台名称",
      "list": [
        {
          "rank": 1,
          "title": "热点标题",
          "hot": "热度数值",
          "url": "跳转链接"
        }
      ]
    }
  ],
  "updateTime": "更新时间"
}
```

### 3\.3 缓存机制

- 各平台数据独立缓存，互不影响

- 缓存TTL：5\-10分钟（可配置）

- 页面展示：更新于 x 分钟前

- 缓存策略：未过期读取缓存，过期重新拉取最新数据并更新缓存

### 3\.4 异常处理

- 单一平台接口失败、数据拉取异常，仅当前卡片失效，不影响其他平台展示

- 异常卡片展示友好提示文案，提升用户体验

- 接口超时自动兜底，避免页面空白报错

### 3\.5 部署要求

- 代码托管至GitHub，目录结构规范

- 全站HTTPS，公网可正常访问

## 四、UI 设计规范

### 4\.1 整体风格

参考今日头条设计：简洁现代、信息分区清晰、信息密度高、无冗余装饰，以内容展示为核心。

### 4\.2 布局规范

- 桌面端：三列卡片网格布局

- 移动端：单列卡片自适应布局

### 4\.3 交互规范

- 榜单标题支持点击跳转

- 卡片悬浮轻微动效，提升页面质感

- 文字超长自动省略，保证页面布局整洁

## 六、非功能需求

### 6\.1 兼容性

兼容主流PC、移动端浏览器，页面无布局错乱、功能异常问题。

### 6\.2 合规说明

本项目为技术学习演示项目，非商用，数据来源于各平台公开榜单，页面标注免责声明。

### 6\.3 性能要求

- 页面首屏加载速度快，渲染流畅

- 缓存机制保障接口响应高效

## 七、测试要点

- **布局测试**：验证电脑端、手机端响应式布局展示正常

- **功能测试**：榜单数据展示、标题点击、更新时间展示正常

- **异常测试**：单平台接口报错隔离生效，友好提示正常展示

- **缓存测试**：缓存时效、更新时间刷新逻辑正常

- **部署测试**：公网HTTPS访问正常，代码完整可用
