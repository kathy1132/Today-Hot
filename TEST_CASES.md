## 3. 测试用例详细说明

### 3.1 TC-001: 健康检查接口

**测试步骤:**
1. 发送 GET 请求到 `/api/health`
2. 验证响应状态码和响应体

**预期结果:**
- HTTP状态码: 200
- 响应体: `{ "ok": true }`

### 3.2 TC-002: 获取微博热榜

**测试步骤:**
1. 发送 GET 请求到 `/api/hot/weibo`
2. 验证响应结构

**预期结果:**
- HTTP状态码: 200
- 响应体包含:
  - `code: 200`
  - `data` 数组包含微博平台数据
  - 数据结构符合 HotPlatform 定义

### 3.3 TC-005: 获取所有平台热榜

**测试步骤:**
1. 发送 GET 请求到 `/api/hot`
2. 验证响应结构

**预期结果:**
- HTTP状态码: 200
- `platforms` 数组包含3个平台数据
- 每个平台数据结构正确

### 3.4 TC-007: 缓存机制验证

**测试步骤:**
1. 第一次请求 `/api/hot/weibo`，记录 `updatedAt`
2. 立即第二次请求 `/api/hot/weibo`
3. 比较两次响应的 `updatedAt`

**预期结果:**
- 两次响应的 `updatedAt` 相同（使用缓存）
- 响应消息包含「缓存」字样

### 3.5 TC-009: 缓存刷新

**测试步骤:**
1. 发送 POST 请求到 `/api/hot/refresh`
2. 验证响应
3. 再次请求 `/api/hot/weibo`，验证 `updatedAt` 更新

**预期结果:**
- POST 请求返回成功
- 后续请求的 `updatedAt` 为新时间

### 3.6 TC-018: HotCard加载状态

**测试步骤:**
1. 渲染 HotCard 组件，设置 loading=true
2. 验证显示内容

**预期结果:**
- 显示加载中提示或骨架屏
- 不显示列表内容

### 3.7 TC-020: HotCard错误状态

**测试步骤:**
1. 渲染 HotCard 组件，设置 error=true, message="测试错误"
2. 验证显示内容

**预期结果:**
- 显示错误信息 "测试错误"
- 显示「点击重试」按钮
- 不显示列表内容
