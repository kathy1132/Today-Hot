## 一、项目概述

### 1\.1 项目简介

今日热搜是一款**多平台热搜聚合Web项目**，采用前后端分离架构，聚合微博、知乎、B站三大平台公开热搜数据。后端统一拉取、格式化、缓存榜单数据，前端响应式渲染卡片视图，实现一站式全网热点浏览，同时作为标准全栈学习实战项目。

### 1\.2 技术设计目标

- 结构清晰：标准化分层架构，解耦请求、数据、业务、视图

- 稳定可靠：独立缓存、平台隔离异常、容错兜底机制

- 可扩展：支持快速新增热搜平台、迭代新功能

- 轻量化：零成本数据源、低服务开销、适配免费托管部署

### 1\.7 设计约束

- 禁止HTML页面爬虫，仅使用各平台公开JSON接口拉取数据

- 不接入平台OAuth授权体系，保证轻量化、零门槛部署

- 单平台故障隔离，不影响整体页面展示

## 四、核心数据模型（TS 类型）

```typescript
interface HotPlatform {

  error?: boolean;      // 是否拉取失败
  message?: string;     // 失败提示文案
}
```

## 五、缓存设计

### 5\.1 缓存介质

服务端内存 Map 缓存，轻量无依赖，适配免费部署服务

### 5\.2 缓存策略

- 缓存粒度：**各平台独立缓存**（weibo / zhihu / bilibili 互不干扰）

- TTL 时效：300～600 秒（5\~10分钟，可配置）

- 缓存内容：完整 HotPlatform 结构化数据

- 更新规则：过期自动清空，下次请求重新拉取并写入新缓存

### 5\.3 前端展示

前端接收后端返回的 `updateTime`，渲染「数据更新于 x 分钟前」

## 七、核心业务流程

### 7\.1 整体请求链路

1. 用户访问首页，前端发起 `/api/hot` 请求

2. 后端优先读取内存缓存

3. 缓存未命中 / 已过期：后端分别请求三方公开JSON接口

4. 后端标准化格式化数据为统一数据模型

5. 写入内存缓存、记录更新时间

6. 组装数据并返回前端

7. 前端遍历渲染多平台 HotCard 卡片

### 7\.2 异常隔离流程

- 三个平台数据拉取**独立 try/catch**

- 单个平台超时/接口失效/数据解析失败，仅标记当前平台 error:true

- 正常平台数据正常返回、正常渲染

- 前端捕获 error 状态，展示友好失败兜底文案
